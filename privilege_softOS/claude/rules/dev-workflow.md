## Development Workflow

### Pre-Dev Gate (ДО написания кода)
1. Проверь аналоги — нет ли уже решения в проекте или зависимостях
2. Stack compatibility — новая зависимость совместима с текущим стеком?
3. Duplication check — не дублируешь ли существующий код?

### API Contract First
- Определи types/interfaces ПЕРЕД реализацией
- Зафиксируй контракт в коде, потом реализуй

### Test-First
- Критичные пути: тесты ДО кода (TDD)
- Остальное: тесты ВМЕСТЕ с кодом
- Никогда "потом допишу тесты"

### Error Budget
- 2 попытки на подход, потом пересмотр архитектуры
- Не повторяй провалившийся подход — ищи альтернативу

### Checkpoints
- `checkpoint:` commit ПЕРЕД каждым крупным изменением
- Rollback должен быть возможен за 1 команду

### Architecture Decisions
- Новое решение → записать в DECISIONS.md (ПОЧЕМУ выбрано)
- Перед работой → прочитать DECISIONS.md (не повторять тупики)
