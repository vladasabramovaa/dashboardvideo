## Documentation Conventions

- `CLAUDE.md` — архитектура, constraints, known issues. Обновляй при изменении архитектуры
- `DECISIONS.md` — ПОЧЕМУ выбрано решение. Записывай каждое нетривиальное решение
- `HOTPATHS.md` — критичные пути, требуют двойного аудита при изменении
- `TODO.md` — текущие задачи с приоритетами. Обновляй при завершении/добавлении задач
- `DEVPLAN.md` — план разработки: фазы, milestones, зависимости, текущая фаза. Обновляй при смене фазы
- НЕ создавай README.md и *.md без явного запроса пользователя
- Документация обновляется ВМЕСТЕ с кодом, не отдельным этапом
