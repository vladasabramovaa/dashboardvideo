## API Cost Guard

> Потерял деньги на cloud compute. НИКОГДА БОЛЬШЕ.

### Перед КАЖДЫМ запуском платного compute (AWS, GCP, fal.ai и др.):
1. Проверь dashboard — текущий баланс, скажи цифру
2. Проверь что файл РЕАЛЬНО изменён — `cat` критичные строки
3. Проверь нет ли запущенных jobs — dashboard / CLI
4. Считай по РЕАЛЬНЫМ ценам — зайди на pricing page, НЕ гадай
5. Скажи пользователю ТОЧНУЮ цену перед запуском
6. Один запуск = один вариант — проверь работу, потом остальные
7. Мониторь первые 2 минуты

### Batch операции (AI API, cloud services):
- Оцени стоимость всего batch ДО запуска
- Начни с 1-2 items для проверки
- Покажи cost estimate пользователю

**Запрещено:** compute без dashboard, оценки "на глаз", параллельные варианты без проверки.
