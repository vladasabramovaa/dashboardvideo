## Evidence Grading — Шкала валидации выводов

> Каждый вывод, который мы используем в рассуждениях, ДОЛЖЕН иметь grade. Не путай гипотезу с фактом.

### Шкала

| Grade | Название | Критерий | Маркер в тексте |
|-------|----------|----------|-----------------|
| **E1** | Факт | Подтверждено в production ИЛИ из первоисточника (official docs, API response, pricing page). Код работает в бою | `[E1]` |
| **E2** | Верифицировано | Воспроизводимо в тестах/бенчмарках. Несколько независимых источников согласны. Код написан, тесты проходят, но НЕ production | `[E2]` |
| **E3** | Синтетика | Результаты на синтетических/тестовых данных. Модель обучена, но не валидирована на реальных данных. Контролируемый бенчмарк | `[E3]` |
| **E4** | Экспертная оценка | Анализ документации/кода без запуска. Экстраполяция. Литературный консенсус (не наш опыт). Cost/time estimates | `[E4]` |
| **E5** | Гипотеза | Теоретическое предположение. Мат. модель без реализации. Аналогия с другой системой | `[E5]` |
| **E6** | Спекуляция | Один непроверенный источник. Устаревшие данные (>6 мес). Предположение без данных | `[E6]` |

### Правила употребления

1. **При рассуждении** — если вывод используется как аргумент, указывай grade:
   - "Service X latency 5ms [E2-benchmark, 100 tests]" — верифицировано в тестах
   - "Cloud provider autoscaling to 1000 nodes [E1-docs]" — факт из первоисточника
   - "Estimated throughput 10K RPS [E4-литература]" — не наш опыт

2. **При принятии решений** — минимальный grade зависит от цены ошибки:
   - Архитектурное решение → требует E1-E2
   - Выбор стека/инструмента → E2-E3 допустимо
   - Research direction → E4-E5 допустимо
   - Финансовые решения (compute spend) → ТОЛЬКО E1

3. **Повышение grade:**
   - E5→E4: найти 2+ независимых источника
   - E4→E3: запустить бенчмарк/тест
   - E3→E2: валидировать на реальных данных ИЛИ 3+ источника
   - E2→E1: production deployment ИЛИ official source confirmation

4. **Понижение grade:**
   - Данные старше 6 мес без перепроверки → grade -1
   - Единственный источник → max E4
   - Наш собственный бенчмарк без внешнего подтверждения → max E3

5. **Synthetic-to-real gap** — ВСЕГДА указывать для ML/AI выводов:
   - Какие данные использованы (synthetic/test/real)
   - Известный gap (если есть): "synthetic accuracy 90%, expected real ~70% [E3, gap ~22%]"

### Запрещено

- Использовать E5-E6 вывод как аргумент для финансового решения
- Писать "X работает" без grade (подразумевает E1, может быть E3)
- Игнорировать synthetic-to-real gap в ML выводах
- Цитировать наши собственные research notes как E1 (они E3-E4 пока не подтверждены)

### В документации (DECISIONS.md, vault notes)

При записи выводов маркируй inline:
```
- Service X inference 1.3μs on M3 Pro [E2-benchmark, 299 tests]
- ML model accuracy 85% on test data [E3-synthetic, gap ~20%]
- Cloud provider autoscaling to 1000 containers [E1-docs]
- Projected system yield 19%/yr [E5-math model, no implementation]
```
