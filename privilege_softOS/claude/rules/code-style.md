## Code Style — Constructor Pattern

- 1 файл = 1 класс = 1 ответственность
- Файл >200 строк → decompose на 2+ кубика
- Функция >30 строк → split на подфункции
- Types/interfaces определяются ДО реализации
- Тесты пишутся ДО или ВМЕСТЕ с кодом, никогда "потом"
- Запрещено: миксины, абстрактные фабрики, DI-контейнеры, слои абстракции
- Новые кубики создаются по тем же паттернам что существующие (self-extension)
- При аудите: если "проблема" = часть паттерна конструктора — это НЕ проблема
